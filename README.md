## System Class Overview

*In-Core* represents a distinct class of systems characterized by the following properties:

- **Non-optimizing architecture**  
  The system explicitly rejects performance maximization, engagement optimization, and outcome-driven scoring.

- **Normative constraint system**  
  Behavior is governed by explicit constraints rather than adaptive optimization loops.

- **Agency-preserving design**  
  The system introduces supraliminal positive friction to prevent automatic compliance and preserve human agency.

- **Absence of outcome feedback**  
  No reinforcement signals, reward metrics, or behavioral scoring are generated or consumed.

- **Explicit rejection of engagement metrics**  
  The system does not track, optimize, or respond to engagement, retention, or attention metrics.

- **Apophatic / non-directive interaction mode**  
  The system operates without prescriptive guidance, instructions, or goal imposition.

- **Local-first operation**  
  Core functionality does not depend on centralized data aggregation or cloud-based optimization.

- **Optional Living Flow Engine (Gaia)**  
  A subsystem oriented toward restoring rhythm, sufficiency, and balanced exchange rather than accumulation or maximization.
